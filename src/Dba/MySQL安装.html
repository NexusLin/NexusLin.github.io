🌂 3、Mysql配置my.ini文件 ​

如果我们启动失败了，就需要去配置我们的my.ini文件了，win11在整个过程之中我看是需要配置的

① 在D:\anzhuang\MySQL\bin【刚刚你存放MySql的】目录下新增加一个配置文件my.ini
② 在D:\anzhuang\MySQL\bin目录下创建一个data文件夹(用于存放数据库数据)

JS
> D:\anzhuang\MySQL\data  
  D:\anzhuang\MySQL\my.ini 
>

👉 my.ini文件配置详解 ​

my.ini文件里面包含以下东西

plain

[mysqld]
#设置服务端使⽤用的字符集为utf-8
character-set-server=UTF8MB4
#绑定IPv4地址
bind-address=0.0.0.0
#设置mysql的端⼝口号
port = 3306
#设置mysql的安装⽬目录
basedir=F:/mysql-5.7.23
#设置mysql数据库的数据的存放⽬目录
datadir=C:/mysql-5.7.23/data
#允许最⼤大连接数
max_connections=2000
#创建新表时将使⽤用的默认存储引擎
default-storage-engine=INNODB
#设置mysql以及数据库的默认编码
[mysql]
default-character-set=utf8
[mysql.server]
default-character-set=utf8
#设置客户端默认字符集
[client]
default-character-set=utf8

特别注意： 这里需要删除所有# 所在的汉语和文字，最后效果如下 下面是详细的my.ini文件配置，根据自己的安装路径进行修改即可

plain
[mysqld]
character-set-server=UTF8MB4
bind-address=0.0.0.0
port = 3306
basedir=D:\anzhuang\MySQL
datadir=D:\anzhuang\MySQL\data
max_connections=2000
default-storage-engine=INNODB
[mysql]
default-character-set=utf8
[mysql.server]
default-character-set=utf8
[client]
default-character-set=utf8

👉特别注意，data文件不为空会报错： ​

🌂Mysql报错以及处理 ​

errno: 1251 Client does not support authentication protocol requested by server; consider upgrading MySQL client ​

javascript
errno: 1251,
sqlMessage: 'Client does not support authentication protocol requested by server; consider upgrading MySQL client',
sqlState: '08004',
fatal: true

原因 ​

中文意思就是：客户端不支持服务器请求的身份验证协议，考虑升级mysql客户端
这里的根本原因是你安装了8.0版本以上的MySQL，密码加密方式发生了变化所以低版本的navicat不能进行不能自动适配，所以简单的方法就是升级navicat客户端，或者是去手动修改密码规则。

解决方案 ​

首先这里不要怀疑是自己记错了密码，和这个完全没有关系

找到你的MySQL安装路径，进入bin目录下
输入cmd进行命令行
输入命令mysql -u root -p
输入密码

更改密码，刷新权限

plain
# 这里最后的root就是你修改后的密码，根据自己的需求更改（假如密码是123456）
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '123456'; 
# 刷新权限
FLUSH PRIVILEGES;

出现了Query OK, 0 rows affected也就成功了

JS
Query OK, 0 rows affected (0.01 sec)
