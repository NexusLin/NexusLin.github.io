Mysql使用 ​

MySQL数据类型 ​

TEXT： 用于存储较大文本。最大存储容量为 65,535 字节（64KB）。如果文本内容不大，使用 TEXT 类型是一个合适的选择。

MEDIUMTEXT： 用于存储中等大小的文本。最大存储容量为 16,777,215 字节（16MB）。适用于中等大小的文本，如文章或日志内容。

LONGTEXT： 用于存储非常大的文本。最大存储容量为 4,294,967,295 字节（4GB）。适合存储大篇幅的文本内容，如长篇文章、评论或公告。

❤️Mysql常用命令 ​

js
//检查版本
mysql --version

//登录Mysql
mysql -u root -p

//启动服务 
net start mysql  

// 停止MySQL服务
net stop mysql 

//卸载mysqll服务
mysqld --remove

//安装mysqll服务
mysqld --install

// 降低 MySQL 服务器的安全认证协议要求
[mysqld]
default_authentication_plugin=mysql_native_password

ALTER USER 'root'@'hostname' localhostWITH mysql_native_password BY '123456';

// 查看用户
SELECT user,host FROM mysql.user;


// 修改密码
ALTER USER 'root'@'localhost' IDENTIFIED BY '123456';


// 初始化mysql
mysqld --initialize --console

执行后，会生成一个临时密码，在控制台输出中可以看到临时密码，将其保存起来，然后就可以使用.

连表查询 ​

场景1：查询活动表的时候同时也查询出对应活动参加的用户status为approved（就是通过申请）的用户信息

JS
SELECT
    a.id AS activity_id,
    a.title AS activity_title,
    a.description AS activity_description,
    a.location AS activity_location,
    a.start_time AS activity_start_time,
    a.end_time AS activity_end_time,
    a.create_time AS activity_create_time,
    a.update_time AS activity_update_time,
    a.activity_type AS activity_type,
    u.id AS user_id,
    u.user_id AS applicant_user_id,
    u.status AS application_status,
    u.applied_time AS application_applied_time
FROM
    activities a
JOIN
    applications u ON a.id = u.activity_id
WHERE
    u.status = 'approved';

最后我们的查询就应该是

JS
SELECT
    a.id AS activity_id,
    a.title AS activity_title,
    a.description AS activity_description,
    a.location AS activity_location,
    a.start_time AS activity_start_time,
    a.end_time AS activity_end_time,
    a.create_time AS activity_create_time,
    a.update_time AS activity_update_time,
    a.activity_type AS activity_type,
    JSON_ARRAYAGG(
        JSON_OBJECT(
            'user_id', u.user_id,
            'applicant_user_id', u.id,
            'application_status', u.status,
            'application_applied_time', u.applied_time
        )
    ) AS userList
FROM
    activities a
JOIN
    applications u ON a.id = u.activity_id
WHERE
    u.status = 'approved'
GROUP BY
    a.id;

接下来我们继续难度升级，借助查询A表时候查询出来的B表中的用户id查询出C表中对应得用户信息

JS
SELECT
    a.id AS activity_id,
    a.title AS title,
    a.description AS description,
    a.location AS activity_location,
    a.start_time AS activity_start_time,
    a.end_time AS activity_end_time,
    a.create_time AS createTime,
    a.update_time AS updateTime,
    a.activity_type AS activity_type,
        JSON_ARRAYAGG(
            JSON_OBJECT(
                'userId', u.user_id,
                'status', u.status,
                'activityId', u.activity_id,
                'applicant_user_id', u.id,
                'appliedTime', u.applied_time,
                'userEmail', u2.id,
                'name',u2.name,
                'avatar',u2.avatar,
                'sex',u2.sex,
                'age',u2.age
            )
        ) AS userList
    FROM
        activities a
    JOIN
        applications u ON a.id = u.activity_id
    JOIN
        user u2 ON u.user_id = u2.id
    WHERE
        u.status = 'approved' AND a.activity_type = ?
    GROUP BY
        a.id;
